# ПРОМПТ: Технический прототип личного кабинета маркетплейса TUNNER / TNR

## Контекст проекта
Разрабатываю технический прототип личного кабинета для B2C-маркетплейса 
спортивного и функционального питания «Фуд Инновация» (бренды TUNNER и TNR). 
Прототип нужен как базовый каркас под дальнейшую интеграцию в 1С-Битрикс 
(редакция «Бизнес»). Ключевая фишка продукта — персонализация через 
тестирование пользователя и выдача индивидуальных планов (добавки, питание, 
тренировки), а также цифровая подписка на закрытый контент.

## Стек
- React 18 (функциональные компоненты, hooks)
- Tailwind CSS (utility-first, без кастомного CSS где возможно)
- React Router v6 (навигация между разделами ЛК)
- Lucide React (иконки)
- Recharts (графики прогресса, КБЖУ, бонусы)
- Все данные — моковые (JSON-объекты в отдельном файле /src/mocks/)
- Структура файлов БЭМ-подобная на уровне компонентов (Header, Sidebar, 
  Card__title, Card__content и т.д. — через data-атрибуты или именование props)

## Архитектура проекта
/src
  /components
    /layout       — Sidebar, Header, PageWrapper
    /ui           — Button, Card, Badge, ProgressBar, Modal, Tabs
    /widgets      — переиспользуемые блоки (StatCard, WorkoutCard и т.п.)
  /pages          — страницы разделов ЛК
  /mocks          — моковые данные (user.js, workouts.js, nutrition.js, 
                    subscription.js, bonuses.js, orders.js, testResults.js)
  /hooks          — useUser, useSubscription и т.д.
  /utils          — форматтеры (currency, dates)
  App.jsx
  main.jsx

## Макет ЛК
Двухколоночный layout:
- **Левая колонка (Sidebar, 260px):** логотип, аватар+имя юзера, 
  навигация по разделам, кнопка выхода
- **Основная область:** хедер с breadcrumbs + уведомлениями + поиском, 
  ниже — контент страницы

## Обязательные разделы (страницы)

### 1. Дашборд (главная ЛК) — /dashboard
- Приветствие по имени
- 4 карточки-метрики: активная подписка, бонусы (баланс + сгорание), 
  текущая программа, прогресс тренировок за неделю
- Виджет «Сегодня»: план питания на день (КБЖУ-прогресс) + ближайшая тренировка
- Быстрые действия: «Пройти тест», «Заказать повтор», «Связаться с нутрициологом»

### 2. Тестирование и результаты — /test
- Если тест не пройден: CTA-блок с описанием и кнопкой «Начать тестирование»
- Мастер прохождения теста (stepper, 5-7 шагов с моковыми вопросами: 
  цели, пол/возраст/вес/рост, уровень активности, ограничения по питанию, 
  предпочтения)
- Progress bar сверху, кнопки «Назад / Далее»
- Финальный экран — «Ваша персональная программа»:
  * Рекомендованные добавки (карточки товаров с CTA «В корзину»)
  * Рекомендованный план питания (ссылка на раздел «Питание»)
  * Рекомендованный план тренировок (ссылка на «Тренировки»)
  * Кнопка «Перепройти тест»
- История тестов (таблица: дата, результат, PDF-экспорт — заглушка)

### 3. Мои тренировки и питание — /workouts, /nutrition
**Тренировки:**
- Вкладки: «Текущая программа» / «История» / «Видео-библиотека»
- Карточка программы: название, длительность курса, прогресс (X из Y тренировок)
- Список тренировок на неделю (день — упражнения — статус «выполнено/пропущено»)
- Детальная страница тренировки: список упражнений с подходами, видео-превью 
  (заглушка-плейсхолдер), таймер между подходами
- Chart прогресса (recharts): объём нагрузки по неделям

**Питание:**
- Вкладки: «План на сегодня» / «Рецепты» / «Дневник»
- Круговые диаграммы КБЖУ (факт/цель)
- Список приёмов пищи (завтрак/обед/ужин/перекусы) с рецептами 
  и кнопкой «Отметить съеденным»
- Рецепты: карточки с КБЖУ, временем готовки, фильтром по цели

### 4. Подписка и контент — /subscription
- Статус подписки: тариф, дата следующего списания, сумма, кнопки 
  «Изменить тариф» / «Отменить» / «Приостановить»
- 3 тарифа (карточки с градиентами, highlight активного):
  * Basic — рецепты + база знаний
  * Pro — + персональные планы + 1 консультация/мес
  * Elite — + безлимитные консультации + биохакинг-гайды
- Библиотека закрытого контента: плитка карточек (Гайды / Рецепты / 
  Интенсивы / Биохакинг) с замочком на неактивных
- Блок «Консультации»: записаться к нутрициологу / тренеру 
  (календарь-заглушка, слоты)
- История платежей (таблица)

### 5. Бонусы и рефералы — /bonuses
- Большая карточка баланса: баллы, сгорит X баллов через Y дней
- Timeline начислений и списаний (история операций)
- Блок «Как получить больше»: регистрация ✓, первый заказ ✓, 
  отзыв +50, день рождения +500 и т.д.
- Реферальная программа:
  * Уникальная ссылка (copy-to-clipboard)
  * Счётчики: приглашено друзей, получено бонусов
  * Таблица приглашённых (имя-замаскировано, статус, бонус)
  * Социальные шаринг-кнопки

### 6. Покупки и заказы — /orders
- Фильтры: «Все / Активные / Доставлены / Отменены»
- Список заказов (карточки): номер, дата, сумма, статус-бейдж, список товаров
- Детальная страница заказа: таймлайн статусов (оформлен → оплачен → 
  собран → отправлен → доставлен), трек-номер, состав заказа, кнопки 
  «Повторить заказ» / «Оставить отзыв»
- Подраздел «Избранное» и «Недавно смотрел» — плитка товаров

### 7. Профиль и настройки — /profile
- Личные данные, адреса доставки, способы оплаты, 
  уведомления (email/SMS/push toggle), смена пароля

## Требования к UI/UX
- **Mobile-first адаптив:** sidebar превращается в burger-меню на <768px
- **Темная/светлая темы** (toggle в хедере, CSS-переменные через Tailwind)
- **Цветовая палитра:** TUNNER — энергичный (предлагаю акцент #FF5A1F 
  оранжевый), TNR — научный (акцент #2E7DFF синий). Основа — нейтральные 
  серые, белый фон
- **Типографика:** Inter для текста, можно акцентный (например, Space Grotesk) 
  для заголовков
- **Анимации:** микроинтеракции на hover, плавные переходы табов 
  (transition-all duration-200), skeleton-loaders при «загрузке» моков 
  (setTimeout имитация)
- **Иконки:** lucide-react единым стилем
- **Карточки:** rounded-2xl, мягкие тени (shadow-sm/md), без тяжёлых границ
- **Пустые состояния:** красивые illustration-placeholder-блоки с CTA

## Моковые данные — пример
Юзер: Анна Иванова, 28 лет, цель «похудение», активная подписка Pro, 
баланс 1250 бонусов, 3 заказа в истории, тест пройден неделю назад, 
текущая программа «Жиросжигание — 8 недель».

## Что нужно сделать
1. Создать полноценную структуру проекта (package.json, vite config, 
   tailwind config, все папки и файлы)
2. Реализовать все 7 разделов с переходами через React Router
3. Все данные подтянуть из /src/mocks/ (симуляция API через async-функции 
   с задержкой 300-500мс для демонстрации loading-состояний)
4. Сделать компоненты максимально переиспользуемыми (Card, Button, 
   Badge — варианты через props)
5. Приложить README с инструкцией запуска (npm i && npm run dev)
6. Код должен быть чистый, с комментариями на русском для ключевых мест

## Что НЕ делать
- Не подключать реальный бэкенд и не писать API-клиент (только моки)
- Не тянуть тяжёлые UI-kit (shadcn/ui, Material — только Tailwind + lucide)
- Не делать авторизацию (юзер всегда «залогинен»)
- Не реализовывать реальные платежи/трекинг — только визуальные заглушки
